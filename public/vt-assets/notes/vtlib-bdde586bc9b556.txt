【打破 AI Agent 的迷思 － Grok 4.20 的 4-Agent 模式的缺陷】我用超過兩年時間研究 AI Agent，得到的結論是：AI Agent 只適合線性行動（一個行動接下一個行動），不適宜平行工作（多個行動同時進行）。簡單來說，就是 AI Agent 的行為習慣和人類太相似。當 AI Agent 各自外出工作，完成後第一件事做甚麼？「開會」：討論各自工作成果分享資訊、溝通、辯論達成共識，提出計劃（然後再「開會」討論下一步計劃）聽起來很合理，但在人類世界，就是因為無止境的「開會」，而令工作永遠無法完成，pre-meeting-meeting, meeting, post-meeting-meeting，沒完沒了。當問題沒有絕對「正確」的解決方案，越多意見，越多衝突，就越難噠到共識。有些朋友入職大公司，工作就是「開會」，到離職的時候，項目還未完成，仍在「開會」，一個普通IT項目，往往要進行五年以上。比較有效的，就是有明確deadline的項目，例如Marketing。　通常會有位「阿姐」（當然有男性，但一般來說阿姐的行動力普遍較強）統籌所有事情，將全部情報掌握手中，亦擔任傳遞情報的工作。「阿姐」開會，不是為了討論和分享情報，阿姐是來收取匯報，然後指派工作。溝通不過是「阿姐」為了釐清自己對事情的理解，沒有打算要同事互相討論，更不需要辯論，因為「阿姐」自己會做決定。Grok的4-Agent 架構，包括：Boss Agent - GrokResearch Agent - Harper (Agent 1)Logic Agent - Benjamin (Agent 2)Creative Agent - Lucas (Agent 3)問題出在，Grok不是「阿姐」，會任由手下四個Agent互相辯論，即是「開會」。所以很多時候，我們會見到片段顯示，三個Agent在無止境地開會，而作為Boss的Grok，就在等他們達到共識，然後無限loop。所以在目前技術來說，線性進行Multi-Agent工作，會比平行工作更有效，Boss Agent將所有資訊自己掌握，不用開會，直接指派工作，就不會令Agent進入「開會無限loop」的深淵。再進一步，就是Boss Agent的手下，都是專門訓練，在不同領域獨當一面的專業Model，比起「四通才一起開會」更有意思。我自己的做法，是每個Agent都用不同供應商，即GPT, Gemini, Claude, 和Grok，就著不同特性指派不同工作。而Boss Agent，我就會用當時能力最強的Agent擔任，比如現在會用 Claude Opus 4.6 Thinking，其餘三個Agent，不用開會只需匯報和接受新任務就可以了。
